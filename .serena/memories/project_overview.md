Purpose: repo for scanning domains for public MCP endpoints and building a source-backed prospecting pipeline for digital-first companies without MCP. Tech stack: Python CLI orchestration under app/, Ruby pipeline scripts under app/ruby, CSV artifacts in data/, SQLite database mcp_scans.db, Docker Compose used for Ruby pipeline runtime. Structure: app/cli.py is the main operator surface; app/commands contains command modules; app/python contains Python ingestion/discovery modules; app/ruby contains scanning/build scripts; data/processed holds rolling intermediate artifacts; data/master and data/final hold canonical datasets; docs/plans holds design docs. Patterns: keep runtime code under app/, keep root shims minimal, prefer explicit latest artifact paths and command-based workflows over ad hoc scripts.
